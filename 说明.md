# Docker Nginx 反向代理标准部署方案

本文档记录一套全新的 **Docker 容器化 Nginx 反向代理部署方案**，并对应仓库中的 `install.sh` 一键初始化脚本。

适用场景：

- Ubuntu 24.04 VPS 初始配置
- 系统首次 `apt update && apt upgrade`
- 使用 Docker 部署反向代理
- 使用 Nginx 作为统一入口
- 多个业务容器通过反代访问
- 可选安装 3x-ui Docker 容器
- 可选安装 CLIProxyAPI Docker 容器
- 配置 SSH 公钥登录、SSH 端口和密码登录策略
- 证书统一放在宿主机
- Nginx 容器只读挂载证书
- 后续便于扩展多个域名、多个服务

## 一、最终标准

```text
项目目录：/opt/reverse-proxy
容器名称：reverse-proxy
Docker 网络：proxy-net
Nginx 配置目录：/opt/reverse-proxy/nginx/conf.d
SSL 证书目录：/opt/reverse-proxy/ssl
日志目录：/opt/reverse-proxy/logs
3x-ui 目录：/opt/3x-ui
CLIProxyAPI 目录：/opt/cli-proxy-api
SSH 公钥脚本：setup-ssh-key-login.sh
```

反向代理服务本体放在 `/opt/reverse-proxy`，证书和配置由宿主机统一管理，通过 volume 只读挂载进 Nginx 容器。业务容器统一接入 `proxy-net`，对外只暴露反代容器的 `80` 和 `443` 端口。

所有容器均使用 `restart: unless-stopped`，脚本会执行 `systemctl enable --now docker`，服务器重启后 Docker 服务和容器会自动启动。

## 二、目录结构

```text
/opt/reverse-proxy/
├── docker-compose.yml
├── nginx/
│   ├── nginx.conf
│   ├── conf.d/
│   │   ├── 00-default.conf
│   │   └── example.com.conf
│   └── templates/
│       └── https-site.conf.template
├── ssl/
│   └── example.com/
│       ├── fullchain.pem
│       └── privkey.pem
└── logs/
```

说明：

- `docker-compose.yml`：Docker Compose 编排文件
- `nginx/nginx.conf`：Nginx 主配置
- `nginx/conf.d/`：每个域名一个站点配置文件
- `nginx/templates/`：站点配置模板
- `ssl/`：SSL 证书目录
- `logs/`：Nginx 访问日志和错误日志

## 三、Ubuntu 24.04 一键初始化

你使用 Xshell 管理多台 Ubuntu 24.04 服务器时，推荐把本仓库发布到 GitHub，然后在每台 VPS 上执行同一条远程安装命令。

示例 URL 使用当前仓库名 `47alan/VPS_simple`，如果你的仓库地址不同，需要替换成自己的 raw 地址：

如果你不想上传 GitHub，也可以只保存在本地，然后上传到服务器目录执行，见下一节。

推荐新服务器先进入菜单，按序号选择安装内容：

```bash
sudo apt-get update -y && sudo apt-get install -y curl ca-certificates && \
curl -fsSL https://raw.githubusercontent.com/47alan/VPS_simple/main/install.sh -o /tmp/reverse-proxy-install.sh && \
sudo bash /tmp/reverse-proxy-install.sh menu
```

菜单可按序号执行：

- 安装基础环境 + Nginx 反代
- 安装 3x-ui
- 安装 CLIProxyAPI
- 全套安装
- 添加 HTTPS 反代站点
- 查看状态、更新镜像
- 停止/卸载容器，默认保留配置和数据

```bash
sudo apt-get update -y && sudo apt-get install -y curl ca-certificates && \
curl -fsSL https://raw.githubusercontent.com/47alan/VPS_simple/main/install.sh -o /tmp/reverse-proxy-install.sh && \
sudo bash /tmp/reverse-proxy-install.sh install
```

这条命令会自动完成：

- 执行 `apt-get update -y`
- 执行 `apt-get upgrade -y`
- 执行 `apt-get autoremove -y`
- 安装 Docker 和 Docker Compose 插件
- 创建 `/opt/reverse-proxy` 标准目录
- 创建 Docker 网络 `proxy-net`
- 写入 `docker-compose.yml`
- 写入 `nginx/nginx.conf`
- 写入 `nginx/conf.d/00-default.conf`
- 启动 `reverse-proxy` 容器
- 执行 `docker exec reverse-proxy nginx -t` 检查配置

如果要一条命令安装反代、3x-ui 和 CLIProxyAPI：

```bash
sudo apt-get update -y && sudo apt-get install -y curl ca-certificates && \
curl -fsSL https://raw.githubusercontent.com/47alan/VPS_simple/main/install.sh -o /tmp/reverse-proxy-install.sh && \
sudo INSTALL_3XUI=1 INSTALL_CLIPROXY=1 bash /tmp/reverse-proxy-install.sh install
```

如果某台服务器不想执行系统升级，可以加：

```bash
sudo SYSTEM_UPGRADE=0 bash /tmp/reverse-proxy-install.sh install
```

## 四、本地文件上传到 VPS

如果脚本只保存在你的电脑本地，不上传 GitHub，推荐上传到服务器的固定目录：

```text
/root/vps-init
```

方式一：用 Xftp 上传：

1. 在 Xshell 里创建目录：

```bash
mkdir -p /root/vps-init
```

2. 用 Xftp 上传这些文件：

```text
install.sh
setup-ssh-key-login.sh
README.md
说明.md
```

3. 在 Xshell 里运行：

```bash
cd /root/vps-init
chmod +x install.sh setup-ssh-key-login.sh
sudo bash ./install.sh menu
```

方式二：在 Windows PowerShell 里运行本仓库的上传脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\upload_to_vps.ps1 -HostName 你的服务器IP -User root -Port 22
```

上传完成后，在 Xshell 里运行：

```bash
cd /root/vps-init
sudo bash ./install.sh menu
```

如果 SSH 不是 22 端口，把 `-Port 22` 改成实际端口。

## 六、Xshell 脚本管理器

仓库中的 `scripts/` 目录提供多个 Xshell VBS 模板：

```text
scripts/xshell_vps_menu.vbs
scripts/xshell_reverse_proxy_install.vbs
scripts/xshell_3xui_install.vbs
scripts/xshell_cli_proxy_install.vbs
scripts/xshell_setup_ssh_key_login.vbs
scripts/xshell_reverse_proxy_add_site.vbs
```

使用方式：

1. 打开 `.vbs` 文件。
2. 把 `INSTALL_URL` 改成你自己的 GitHub raw 地址。
3. 在 Xshell 连接到目标 Ubuntu 24.04 VPS。
4. 在脚本管理器里运行对应 VBS。

初始化脚本默认只创建反代基础环境，不创建站点：

```vbscript
Const CREATE_SITE = "0"
Const SYSTEM_UPGRADE = "1"
Const INSTALL_3XUI = "1"
Const INSTALL_CLIPROXY = "1"
Const DOMAIN = ""
Const UPSTREAM = ""
```

说明：

- `SYSTEM_UPGRADE="1"` 表示安装前执行系统更新升级。
- `INSTALL_3XUI="1"` 表示同时安装 3x-ui。
- `INSTALL_CLIPROXY="1"` 表示同时安装 CLIProxyAPI。
- 如果只想安装 Nginx 反代，把 `INSTALL_3XUI` 和 `INSTALL_CLIPROXY` 改成 `"0"`。

如果要第一次安装时顺便创建站点，可以改成：

```vbscript
Const CREATE_SITE = "1"
Const DOMAIN = "example.com"
Const UPSTREAM = "app-container:8080"
```

注意：站点启用前必须先把证书放到：

```text
/opt/reverse-proxy/ssl/example.com/fullchain.pem
/opt/reverse-proxy/ssl/example.com/privkey.pem
```

如果证书不存在，脚本只会生成 `.conf.disabled`，不会让 Nginx 因证书缺失而启动失败。

## 七、SSH 公钥登录和端口处理

新增脚本：

```text
setup-ssh-key-login.sh
```

添加公钥，默认不关闭密码登录：

```bash
sudo SSH_USER=root SSH_PUBLIC_KEY='ssh-ed25519 AAAA...你的公钥...' bash ./setup-ssh-key-login.sh
```

修改 SSH 端口，默认保留旧端口：

```bash
sudo SSH_USER=root SSH_PORT=22222 SSH_PUBLIC_KEY='ssh-ed25519 AAAA...你的公钥...' bash ./setup-ssh-key-login.sh
```

确认新端口和私钥登录成功后，再关闭密码登录并移除旧端口：

```bash
sudo SSH_USER=root SSH_PORT=22222 KEEP_OLD_SSH_PORT=0 DISABLE_PASSWORD_LOGIN=1 SSH_PUBLIC_KEY='ssh-ed25519 AAAA...你的公钥...' bash ./setup-ssh-key-login.sh
```

端口处理原则：

- 不要关闭当前 Xshell 会话。
- 设置 `SSH_PORT` 前，先确认云厂商安全组会放行该 TCP 端口。
- 默认 `KEEP_OLD_SSH_PORT=1`，会同时监听旧端口和新端口，防止误锁。
- 只有确认私钥登录成功后，才设置 `DISABLE_PASSWORD_LOGIN=1`。
- 脚本会先执行 `sshd -t`，配置测试通过后才 reload SSH 服务。

## 八、添加 HTTPS 站点

先放入证书：

```bash
sudo mkdir -p /opt/reverse-proxy/ssl/example.com
sudo cp fullchain.pem /opt/reverse-proxy/ssl/example.com/fullchain.pem
sudo cp privkey.pem /opt/reverse-proxy/ssl/example.com/privkey.pem
sudo chmod 644 /opt/reverse-proxy/ssl/example.com/fullchain.pem
sudo chmod 600 /opt/reverse-proxy/ssl/example.com/privkey.pem
```

再生成站点配置：

```bash
sudo DOMAIN=example.com UPSTREAM=app-container:8080 bash ./install.sh add-site
```

`UPSTREAM` 可以写成：

```text
app-container:8080
http://app-container:8080
https://app-container:8443
```

如果证书还不存在，脚本会生成未启用文件：

```text
/opt/reverse-proxy/nginx/conf.d/example.com.conf.disabled
```

这样可以避免 Nginx 因证书缺失而启动失败。证书放好后重新执行 `add-site`，脚本会生成正式的 `.conf` 并重载 Nginx。

## 九、docker-compose.yml 标准内容

```yaml
services:
  reverse-proxy:
    image: nginx:stable
    container_name: reverse-proxy
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      - ./ssl:/etc/nginx/ssl:ro
      - ./logs:/var/log/nginx
    networks:
      - proxy

networks:
  proxy:
    external: true
    name: proxy-net
```

## 十、Nginx 主配置

```nginx
user  nginx;
worker_processes  auto;

events {
    worker_connections  1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    access_log  /var/log/nginx/access.log;
    error_log   /var/log/nginx/error.log warn;

    sendfile        on;
    keepalive_timeout  65;
    server_tokens off;
    client_max_body_size 100m;

    map $http_upgrade $connection_upgrade {
        default upgrade;
        ''      close;
    }

    include /etc/nginx/conf.d/*.conf;
}
```

## 十一、站点配置示例

```nginx
server {
    listen 80;
    server_name example.com;

    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    http2 on;

    server_name example.com;

    ssl_certificate     /etc/nginx/ssl/example.com/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/example.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;

    location / {
        proxy_pass http://app-container:8080;
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;

        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}
```

需要修改的地方：

- `server_name example.com;` 改成真实域名
- `/etc/nginx/ssl/example.com/` 改成对应证书目录
- `proxy_pass http://app-container:8080;` 改成真实业务容器名和端口

## 十二、业务容器接入反代网络

业务容器需要加入同一个 Docker 网络 `proxy-net`：

```yaml
services:
  app-container:
    image: your-app-image
    container_name: app-container
    restart: unless-stopped
    expose:
      - "8080"
    networks:
      - proxy-net

networks:
  proxy-net:
    external: true
```

推荐使用 `expose` 而不是 `ports`：

```yaml
expose:
  - "8080"
```

原因：

- 后端服务不需要直接暴露到公网
- 只需要让 `reverse-proxy` 容器访问即可
- 对外统一由 Nginx 暴露 `80` 和 `443`

## 十四、安全建议

1. 证书只读挂载：

```yaml
- ./ssl:/etc/nginx/ssl:ro
```

2. 不要把证书写进 Dockerfile：

```dockerfile
# 不推荐
COPY fullchain.pem /etc/nginx/ssl/
COPY privkey.pem /etc/nginx/ssl/
```

3. 私钥权限收紧：

```bash
sudo chmod 600 /opt/reverse-proxy/ssl/example.com/privkey.pem
```

4. 后端服务不要直接暴露公网端口：

```yaml
# 推荐
expose:
  - "8080"
```
